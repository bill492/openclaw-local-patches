# OV-013

## Purpose

OV-013 is the system-prompt runtime-alignment patch for OpenClaw `2026.4.14`.

Use this when the agent system prompt is still teaching outdated messaging or delegation guidance that does not match the real runtime contract.

This document is the reusable operator reference. Another Codex session should be able to read only this file and implement the same patch on another OpenClaw node.

## What It Fixes

OV-013 fixes two prompt mismatches.

### Part 1: `message(action=send)` argument guidance

The prompt should tell the model to use:

- `target`
- `message`

for `message(action=send)`.

If it still says `to` instead of `target`, the prompt is teaching an obsolete contract.

### Part 2: sub-agent orchestration guidance

The prompt should not always tell the model to use only `subagents(action=list|steer|kill)`.

The correct behavior depends on tool availability:

- if `sessions_spawn` exists, use it to start delegated work
- if `subagents` exists, use it to manage already-spawned children
- if both exist, say both
- if only one exists, say only that one

## Why This Exists

This is not a behavioral runtime bug by itself. It is an agent-guidance bug.

If the prompt advertises the wrong tool contract:

- the model can call the wrong argument shape for `message(action=send)`
- the model can misuse `subagents` when it should be starting work with `sessions_spawn(...)`
- tool behavior and prompt behavior drift apart, which creates avoidable operator-facing confusion

## Exact Behavior Change

### Before

- `message(action=send)` guidance said to include `to` and `message`
- sub-agent orchestration guidance was hardcoded to `subagents(action=list|steer|kill)`

### After

- `message(action=send)` guidance says to include `target` and `message`
- sub-agent orchestration guidance is computed from available tools:
  - `sessions_spawn(...)` for starting delegated work
  - `subagents(action=list|steer|kill)` for managing already-spawned children

## Implementation

Locate `buildMessagingSection(params)` in the active `system-prompt-*.js` bundle.

Make these changes:

1. Add:

```ts
const hasSessionsSpawn = params.availableTools.has("sessions_spawn");
const hasSubagents = params.availableTools.has("subagents");
const subagentOrchestrationGuidance =
  hasSessionsSpawn
    ? hasSubagents
      ? "- Sub-agent orchestration → use `sessions_spawn(...)` to start delegated work; use `subagents(action=list|steer|kill)` to manage already-spawned children."
      : "- Sub-agent orchestration → use `sessions_spawn(...)` to start delegated work."
    : hasSubagents
      ? "- Sub-agent orchestration → use `subagents(action=list|steer|kill)` to manage already-spawned children."
      : "";
```

2. Replace the hardcoded sub-agent line with `subagentOrchestrationGuidance`.

3. Change:

```ts
"- For `action=send`, include `to` and `message`."
```

to:

```ts
"- For `action=send`, include `target` and `message`."
```

## Verification

After patching:

1. Restart the gateway.
2. Confirm the active `system-prompt-*.js` bundle contains:
   - `subagentOrchestrationGuidance`
   - `sessions_spawn(...)`
   - `include \`target\` and \`message\``
3. Spot-check a generated system prompt if needed.
