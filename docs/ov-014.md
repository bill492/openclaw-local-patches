# OV-014

## Purpose

OV-014 is the pathological session-export line wrapping patch aligned with the final rebased shape of `openclaw/openclaw#64494`.

Use this when session export still keeps one message per output line even for pathological giant blobs, which makes downstream indexing and QMD-style chunking fragile.

This document is the reusable operator reference. Another Codex session should be able to read only this file and implement the same patch on another OpenClaw node.

## What It Fixes

Normal session export intentionally keeps one line per message:

- `User: ...`
- `Assistant: ...`

That is fine for normal turns.

The problem is pathological long messages. Without wrapping, one toxic giant line can be exported as a single line and handed downstream to indexers or search systems.

OV-014 preserves the historical one-line-per-message shape for normal turns, but wraps pathological long exports into multiple lines while preserving source mapping and timestamp alignment.

## Why This Exists

This is a downstream robustness patch.

Without it:

- a giant user or assistant blob becomes one giant exported line
- downstream indexing and QMD-style consumers have to ingest that line whole
- line-oriented debugging and provenance become worse
- the current export path's per-message timestamp array stops matching the flattened output once wrapping is introduced later

With it:

- normal turns stay unchanged
- pathological turns are wrapped safely
- continuation lines still map back to the same original JSONL source line
- continuation lines also repeat the original per-message timestamp
- hard wraps avoid splitting UTF-16 surrogate pairs
- wrapping stays linear even for giant space-free tokens such as pasted blobs or base64

### Before

The export path did:

```ts

```

### After

The export path:

- defines `SESSION_EXPORT_CONTENT_WRAP_CHARS = 800`
- adds `isHighSurrogate(...)`
- adds `isLowSurrogate(...)`
- splits pathological long lines with `splitLongSessionLine(...)`
- searches for split points only inside the current `[cursor, limit]` window
- renders wrapped output with `renderSessionExportLines(label, safe)`
- repeats `lineMap` entries for each wrapped continuation line
- repeats `messageTimestampsMs` entries for each wrapped continuation line
- avoids splitting surrogate pairs when hard-wrapping long tokens with no spaces

That timestamp repetition is important on the current live bundle shape because the export path already tracks `messageTimestampsMs` alongside `lineMap`.

## Implementation

Locate the session-files section in the active `engine-qmd-*.js` bundle. Mirror the real upstream fix shape from `src/memory-host-sdk/host/session-files.ts`.

### 1. Add the content-only wrap limit

```ts
const SESSION_EXPORT_CONTENT_WRAP_CHARS = 800;
```

### 2. Add the helper functions

```ts
function isHighSurrogate(code) {
  return code >= 0xd800 && code <= 0xdbff;
}

function isLowSurrogate(code) {
  return code >= 0xdc00 && code <= 0xdfff;
}

function splitLongSessionLine(text, maxChars = SESSION_EXPORT_CONTENT_WRAP_CHARS) {
  const normalized = text.trim();
  if (!normalized) return [];
  if (normalized.length <= maxChars) return [normalized];

  const segments = [];
  let cursor = 0;
  while (cursor < normalized.length) {
    const remaining = normalized.length - cursor;
    if (remaining <= maxChars) {
      segments.push(normalized.slice(cursor).trim());
      break;
    }

    const limit = cursor + maxChars;
    let splitAt = limit;
    for (let index = limit; index > cursor; index -= 1) {
      if (normalized[index] === " ") {
        splitAt = index;
        break;
      }
    }
    if (
      splitAt < normalized.length &&
      splitAt > cursor &&
      isHighSurrogate(normalized.charCodeAt(splitAt - 1)) &&
      isLowSurrogate(normalized.charCodeAt(splitAt))
    ) {
      splitAt -= 1;
    }

    segments.push(normalized.slice(cursor, splitAt).trim());
    cursor = splitAt;
    while (cursor < normalized.length && normalized[cursor] === " ") cursor += 1;
  }

  return segments.filter(Boolean);
}

function renderSessionExportLines(label, text) {
  return splitLongSessionLine(text).map((segment) => `${label}: ${segment}`);
}
```

### 3. Replace the single-line export write

Replace:

```ts
const safe = redactSensitiveText(text, { mode: "tools" });
const label = message.role === "user" ? "User" : "Assistant";
collected.push(`${label}: ${safe}`);
lineMap.push(jsonlIdx + 1);
messageTimestampsMs.push(parseSessionTimestampMs(record, message));
```

with:

```ts
const safe = redactSensitiveText(text, { mode: "tools" });
const label = message.role === "user" ? "User" : "Assistant";
const renderedLines = renderSessionExportLines(label, safe);
const timestampMs = parseSessionTimestampMs(record, message);
collected.push(...renderedLines);
lineMap.push(...renderedLines.map(() => jsonlIdx + 1));
messageTimestampsMs.push(...renderedLines.map(() => timestampMs));
```

## What Must Stay Unchanged

Do not disturb the surrounding logic for:

- inbound metadata stripping on user messages
- dreaming-narrative transcript detection and skipping
- existing timestamp parsing behavior
- existing content hashing beyond the natural effect of wrapped output

OV-014 is only the pathological export-line wrapping patch.

## Verification

After patching:

1. Restart the gateway.
2. Confirm the active `engine-qmd-*.js` bundle contains:
   - `SESSION_EXPORT_CONTENT_WRAP_CHARS`
   - `isHighSurrogate`
   - `isLowSurrogate`
   - `splitLongSessionLine`
   - `renderSessionExportLines`
3. Confirm wrapped continuation lines repeat both:
   - `lineMap`
   - `messageTimestampsMs`
4. Confirm hard-wrapped astral Unicode does not produce unpaired surrogate halves.
5. Confirm the split-point scan is bounded to the current chunk window and does not call an unbounded `lastIndexOf(" ", limit)` on every iteration.
