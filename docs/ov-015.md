# OV-015

## Purpose

OV-015 is the Slack mention-token annotation patch for OpenClaw `2026.4.14`.

Use this when inbound Slack bodies still preserve raw mention tokens like `<@U123>` without a human-readable annotation.

This document is the reusable operator reference. Another Codex session should be able to read only this file and implement the same patch on another OpenClaw node.

## What It Fixes

Slack inbound content can contain user mentions like:

- `<@U123>`
- `<@U123|name>`

If those tokens are passed straight through to agent-facing bodies, readability suffers and debugging is worse.

OV-015 annotates those tokens using resolved user names so the body becomes:

- `<@U123> (Alice)`

when the name can be resolved.

If the user lookup fails, the raw mention token is preserved.

## Why This Exists

This is an inbound readability and debugging patch.

Without it:

- agent-visible Slack payloads contain opaque IDs
- user mentions are harder to read in transcripts and debugging output

With it:

- names are added where available
- unresolved mentions stay unchanged
- the patch is safe and best-effort

## Exact Behavior Change

### Before

`resolveSlackMessageContent(...)` built `rawBody` from:

- `params.message.text`
- `attachmentContent?.text`
- `botAttachmentText`

with no mention annotation pass.

### After

`resolveSlackMessageContent(...)`:

- accepts `resolveUserName`
- uses the shared runtime `runTasksWithConcurrency(...)` helper
- resolves unique mentioned Slack user IDs
- rewrites `<@USER>` tokens as `<@USER> (Name)` when the name is available
- keeps the original raw token when lookup fails

Also:

- `prepareSlackMessage(...)` passes `ctx.resolveUserName` into `resolveSlackMessageContent(...)`

## Implementation

Do not add a private helper file for this patch.

Use the shared runtime concurrency helper already present in the current codebase.

Important: the import must be relative to the actual patched bundle location, not copied blindly from another dist seam.

For the Slack prepare bundle under `dist/extensions/slack/prepare-*.js`, the correct shape is:

```ts
import { t as runTasksWithConcurrency } from "../../run-with-concurrency-*.js";
```

Do not use:

```ts
import { t as runTasksWithConcurrency } from "./run-with-concurrency-*.js";
```

That path is wrong for `dist/extensions/slack/*` bundles and will break plugin registration at startup.

Then patch `resolveSlackMessageContent(...)` in the active `prepare-*.js` bundle.

### Step 1: import the shared helper

Add an import for `runTasksWithConcurrency` from the active `run-with-concurrency-*.js` bundle.

Resolve the path relative to the patched file before writing it. For `dist/extensions/slack/prepare-*.js`, that means walking up two levels to the dist root.

### Step 2: add mention resolution logic

Add:

```ts
const SLACK_MENTION_RESOLUTION_CONCURRENCY = 4;
```

and inside `resolveSlackMessageContent(...)` add a helper equivalent to:

```ts
const renderSlackUserMentions = async (text) => {
  if (!text || !params.resolveUserName) return text;
  const mentionIds = Array.from(
    new Set(
      Array.from(
        text.matchAll(/<@([A-Z0-9]+)(?:\|[^>]+)?>/gi),
        (match) => match[1],
      ).filter(Boolean),
    ),
  );
  if (mentionIds.length === 0) return text;

  const seen = new Map();
  const { results } = await runTasksWithConcurrency({
    tasks: mentionIds.map((userId) => async () => {
      const user = await params.resolveUserName?.(userId);
      const renderedName = normalizeOptionalString(user?.name);
      return {
        userId,
        rendered: renderedName ? `<@${userId}> (${renderedName})` : null,
      };
    }),
    limit: SLACK_MENTION_RESOLUTION_CONCURRENCY,
  });

  for (const result of results) {
    if (!result) continue;
    seen.set(result.userId, result.rendered);
  }
  if (seen.size === 0) return text;

  return text.replace(/<@([A-Z0-9]+)(?:\|[^>]+)?>/gi, (full, userId) => {
    const rendered = seen.get(userId);
    return rendered ?? full;
  });
};
```

### Step 3: build `rawBody` from rendered text

Replace direct raw-body assembly with:

```ts
const renderedMessageText = await renderSlackUserMentions(normalizeOptionalString(params.message.text));
const renderedAttachmentText = await renderSlackUserMentions(attachmentContent?.text);
const renderedBotAttachmentText = await renderSlackUserMentions(botAttachmentText);
```

and build `rawBody` from those rendered values.

### Step 4: pass `resolveUserName` from `prepareSlackMessage(...)`

Update the `resolveSlackMessageContent(...)` call to include:

```ts
resolveUserName: ctx.resolveUserName
```
