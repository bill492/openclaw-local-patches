# OV-017

## Purpose

OV-017 is the Slack thread bootstrap context hygiene patch for OpenClaw `2026.4.16`.

Use this on another install when one or both of these are true:

- a fresh Slack thread session replays prior bot-authored thread text back into the prompt
- first-turn Slack thread bootstrap includes the thread starter twice, once as structured untrusted context and again as a plain-text `[Thread starter - for context]` prelude

This document is intentionally self-contained. Another Codex session should be able to read only this file and apply the same patch set.

## What It Fixes

OV-017 has two parts.

### Part 1: stop seeding new Slack thread sessions with bot-authored thread history

On a new Slack thread session, `resolveSlackThreadContextData(...)` can fetch existing thread replies and build `ThreadHistoryBody`.

Before OV-017, that history seeding path could include prior bot-authored Slack replies. That means low-value assistant output already posted into Slack could be replayed back into the next fresh session as agent-visible context.

OV-017 filters bot-authored thread messages out before:

- user-name resolution
- allowlist/context-visibility filtering
- final `ThreadHistoryBody` formatting

The remaining seeded history is human-authored only.

### Part 2: remove the duplicate plain-text thread-starter prelude

When `ThreadHistoryBody` is absent but `ThreadStarterBody` is present, `runPreparedReply(...)` can add:

- the structured untrusted thread-starter block from inbound metadata
- and a second plain-text `[Thread starter - for context]` prelude

That duplicates the same root thread text in the prompt and wastes tokens.

OV-017 keeps the structured untrusted metadata form and removes only the extra plain-text prelude.

## Why This Exists

The problem is not "Slack thread bootstrap should never exist." The problem is narrower:

- new-session thread seeding should preserve useful human context
- bot-authored Slack replies are noisy and can replay low-value assistant text into the next session
- duplicating `ThreadStarterBody` across two prompt surfaces wastes tokens without adding signal

OV-017 patches those exact seams without changing the general first-turn Slack thread bootstrap model.

## Exact Behavior Change

### Part 1 behavior

Before, the Slack thread bootstrap path effectively did:

```ts
const uniqueUserIds = [
  ...new Set(
    threadHistory.map((item) => item.userId).filter((id): id is string => Boolean(id)),
  ),
];

const { items: filteredThreadHistory, omitted: omittedHistoryCount } =
  filterSupplementalContextItems({
    items: threadHistory,
    ...
  });

for (const historyMsg of filteredThreadHistory) {
  const isBot = Boolean(historyMsg.botId);
  const role = isBot ? "assistant" : "user";
  ...
  from: `${msgSenderName} (${role})`,
}
```

After, it should do the equivalent of:

```ts
const threadHistoryWithoutBots = threadHistory.filter((historyMsg) => !historyMsg.botId);
const omittedBotHistoryCount = threadHistory.length - threadHistoryWithoutBots.length;

const uniqueUserIds = [
  ...new Set(
    threadHistoryWithoutBots
      .map((item) => item.userId)
      .filter((id): id is string => Boolean(id)),
  ),
];

const { items: filteredThreadHistory, omitted: omittedHistoryCount } =
  filterSupplementalContextItems({
    items: threadHistoryWithoutBots,
    ...
  });

for (const historyMsg of filteredThreadHistory) {
  ...
  from: `${msgSenderName} (user)`,
}
```

Contract:

- bot-authored Slack thread replies are not seeded into `ThreadHistoryBody`
- `resolveUserName(...)` is only called for surviving human-authored history entries
- seeded Slack thread history is always formatted as user-authored context

### Part 2 behavior

Before:

```ts
const threadStarterBody = normalizeOptionalString(ctx.ThreadStarterBody);
const threadHistoryBody = normalizeOptionalString(ctx.ThreadHistoryBody);
const threadContextNote = threadHistoryBody
  ? `[Thread history - for context]\n${threadHistoryBody}`
  : threadStarterBody
    ? `[Thread starter - for context]\n${threadStarterBody}`
    : undefined;
```

After:

```ts
const threadHistoryBody = normalizeOptionalString(ctx.ThreadHistoryBody);
const threadContextNote = threadHistoryBody
  ? `[Thread history - for context]\n${threadHistoryBody}`
  : undefined;
```

Contract:

- `ThreadHistoryBody` still produces the plain-text thread prelude
- `ThreadStarterBody` remains available via the structured untrusted metadata block
- the extra `[Thread starter - for context]` duplication is removed

## Implementation

### Step 1: patch Slack thread-history seeding in the active `prepare-*.js` bundle

Patch `resolveSlackThreadContextData(...)` so bot-authored thread messages are removed before user lookup and history formatting.

On a matching `2026.4.16` bundle, the target shape will be equivalent to:

```ts
if (threadHistory.length > 0) {
  const uniqueUserIds = [
    ...new Set(
      threadHistory.map((item) => item.userId).filter((id) => Boolean(id)),
    ),
  ];
  ...
  const { items: filteredThreadHistory, omitted: omittedHistoryCount } =
    filterSupplementalContextItems({
      items: threadHistory,
      ...
    });
  ...
  for (const historyMsg of filteredThreadHistory) {
    const msgSenderName = msgUser?.name ?? (historyMsg.botId ? `Bot (${historyMsg.botId})` : "Unknown");
    const isBot = Boolean(historyMsg.botId);
    const role = isBot ? "assistant" : "user";
    ...
    from: `${msgSenderName} (${role})`,
  }
}
```

Replace it with the equivalent logical behavior:

- compute `threadHistoryWithoutBots`
- compute `omittedBotHistoryCount`
- derive `uniqueUserIds` from `threadHistoryWithoutBots`
- pass `threadHistoryWithoutBots` into `filterSupplementalContextItems(...)`
- log `omittedHistoryCount + omittedBotHistoryCount`
- format the surviving history entries as `${msgSenderName} (user)`

If you want an automated patch on a bundle with the same shape, use this:

```bash
python3 - <<'PY'
from pathlib import Path

dist_dir = Path.home() / ".local/share/fnm/node-versions/v24.13.1/installation/lib/node_modules/openclaw/dist"
prepare_files = sorted(dist_dir.glob("prepare-*.js"))

patched = []
for path in prepare_files:
    text = path.read_text()
    original = text

    needle = """if (threadHistory.length > 0) {\n      const uniqueUserIds = [\n        ...new Set(\n          threadHistory.map((item) => item.userId).filter((id) => Boolean(id)),\n        ),\n      ];"""
    replacement = """if (threadHistory.length > 0) {\n      const threadHistoryWithoutBots = threadHistory.filter((historyMsg) => !historyMsg.botId);\n      const omittedBotHistoryCount = threadHistory.length - threadHistoryWithoutBots.length;\n      const uniqueUserIds = [\n        ...new Set(\n          threadHistoryWithoutBots.map((item) => item.userId).filter((id) => Boolean(id)),\n        ),\n      ];"""
    if needle in text:
      text = text.replace(needle, replacement, 1)

    text = text.replace("items: threadHistory,", "items: threadHistoryWithoutBots,")
    text = text.replace(
        "`slack: omitted ${omittedHistoryCount} thread message(s) from context (mode=${params.contextVisibilityMode})`",
        "`slack: omitted ${omittedHistoryCount + omittedBotHistoryCount} thread message(s) from context (mode=${params.contextVisibilityMode})`",
    )
    text = text.replace(
        "const msgSenderName = msgUser?.name ?? (historyMsg.botId ? `Bot (${historyMsg.botId})` : \"Unknown\");\n        const isBot = Boolean(historyMsg.botId);\n        const role = isBot ? \"assistant\" : \"user\";",
        "const msgSenderName = msgUser?.name ?? \"Unknown\";",
    )
    text = text.replace("from: `${msgSenderName} (${role})`,", "from: `${msgSenderName} (user)`,")

    if text != original:
        path.write_text(text)
        patched.append(path)

if not patched:
    raise SystemExit("OV-017 step 1 failed: expected Slack prepare bundle shape not found in any prepare-*.js bundle")

print("OV-017 step 1 patched:")
for path in patched:
    print(path)
PY
```

If the local bundle shape differs, do not force a blind replacement. Find `resolveSlackThreadContextData(...)` and make the equivalent logical change described above.

### Step 2: patch `runPreparedReply(...)` in the active `reply-*.js` bundle

In the active `reply-*.js` bundle, locate the thread-context prelude logic containing:

```ts
`[Thread starter - for context]\\n${threadStarterBody}`
```

Remove only the thread-starter fallback branch. Keep the thread-history branch intact.

On a matching bundle, the target shape will be equivalent to:

```ts
const threadStarterBody = normalizeOptionalString(ctx.ThreadStarterBody);
const threadHistoryBody = normalizeOptionalString(ctx.ThreadHistoryBody);
const threadContextNote = threadHistoryBody
  ? `[Thread history - for context]\n${threadHistoryBody}`
  : threadStarterBody
    ? `[Thread starter - for context]\n${threadStarterBody}`
    : undefined;
```

Replace it with:

```ts
const threadStarterBody = normalizeOptionalString(ctx.ThreadStarterBody);
const threadHistoryBody = normalizeOptionalString(ctx.ThreadHistoryBody);
const threadContextNote = threadHistoryBody
  ? `[Thread history - for context]\n${threadHistoryBody}`
  : undefined;
```

It is fine if the local bundle keeps an unused `threadStarterBody` local. The behavioral change is the removal of the fallback branch.

If you want an automated patch on a bundle with the same shape, use this:

```bash
python3 - <<'PY'
from pathlib import Path

dist_dir = Path.home() / ".local/share/fnm/node-versions/v24.13.1/installation/lib/node_modules/openclaw/dist"
reply_files = sorted(dist_dir.glob("reply-*.js"))

patched = []
for path in reply_files:
    text = path.read_text()
    original = text
    needle = """const threadStarterBody = normalizeOptionalString(ctx.ThreadStarterBody);\n  const threadHistoryBody = normalizeOptionalString(ctx.ThreadHistoryBody);\n  const threadContextNote = threadHistoryBody\n    ? `[Thread history - for context]\\n${threadHistoryBody}`\n    : threadStarterBody\n      ? `[Thread starter - for context]\\n${threadStarterBody}`\n      : undefined;"""
    replacement = """const threadStarterBody = normalizeOptionalString(ctx.ThreadStarterBody);\n  const threadHistoryBody = normalizeOptionalString(ctx.ThreadHistoryBody);\n  const threadContextNote = threadHistoryBody\n    ? `[Thread history - for context]\\n${threadHistoryBody}`\n    : undefined;"""
    if needle in text:
        text = text.replace(needle, replacement, 1)
    if text != original:
        path.write_text(text)
        patched.append(path)

if not patched:
    raise SystemExit("OV-017 step 2 failed: expected reply bundle thread-context shape not found in any reply-*.js bundle")

print("OV-017 step 2 patched:")
for path in patched:
    print(path)
PY
```

If the local bundle shape differs, search for `[Thread starter - for context]` in the active `reply-*.js` bundle and remove only that fallback branch.
